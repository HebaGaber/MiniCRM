### Decisions

- **DEC-UX-1 — Token source reconciliation.** The iSolution Design System and the constitution's "Claude Design tokens" are the **same token layer** (the iSolution DS is delivered as Claude Design tokens, already available on Claude). Resolution: no amendment to §8.4/NFR-10 token *source*; the iSolution DS is the source of all visual values, referenced by token name. *(Heba, 2026-06-06)*
- **DEC-UX-2 — Motion is the one authored layer.** The DS owns color/type/spacing/radius; it does **not** own motion. Motion tokens (duration, easing) are defined in DESIGN.md frontmatter as this layer's only authored visual values. *(Heba)*
- **DEC-UX-3 — Motion level: functional + light polish.** Motion carries meaning first (rollback, saga progress, tone change, skeleton→ready); modest hover/focus/page polish is permitted on top. No decorative motion system. Honors `prefers-reduced-motion`. *(Heba, 2026-06-06)*
- **DEC-UX-4 — Output shape.** Two spines (DESIGN.md thin/by-reference + EXPERIENCE.md rich) + a paste-ready `§8-binding-rules.md` addendum block. The UX run does **not** edit `project-context.md`; Heba folds the block in. *(Heba)*
- **DEC-UX-5 — Activity timeline ≠ Audit log.** The Activity timeline is the per-record *product* surface rendering `DomainEvent`s chronologically (actor + time), gated by view-record permission (viewer sees it in scope). The Audit/Events log is a separate raw compliance view of `AuditEvent` before/after, gated by the §6.2 "View audit/events" row (viewer = none). Two distinct surfaces; never merged. *(architecture E4-S4 / prd line 397)*
- **DEC-UX-6 — Illegal transition is silent on the pill.** A 422 (illegal transition or customer-state gate) NEVER mutates the StatusPill; feedback is inline at the action origin, not a tone change. *(UC-3, Pattern 4)*

## Run: prototype reconciliation 2026-06-07 (correct-course)

> The Claude Design prototype (`prototype/`) is now built and is the realized UX oracle. The
> following decisions reconcile the spine to what the prototype actually settled. Full context:
> `_bmad-output/planning-artifacts/sprint-change-proposal-2026-06-07.md`. Items where the prototype
> *contradicts* the constitution are NOT decided here — they are escalated as conflicts C-1…C-4 in
> that proposal and the constitution wins until ruled on.

- **DEC-UX-7 — OQ3 resolved: the read-model widget is the conversion funnel.** E5-S2's open
  "conversion funnel OR per-subsidiary roll-up" is settled by the prototype to the **conversion
  funnel** — five derived, read-only stages (New · Contacted · Qualified · Converted · Active
  customers), scope-aware, "Derived · read-only". The per-subsidiary roll-up is **not dropped** —
  it remains the separate **E1-S5 Roll-up** surface. Oracle: `prototype/screenshots/dashboard.png`,
  code `prototype/app/dashboard.jsx`. *(supersedes the OQ3 "decide at sprint planning" deferral)*
- **DEC-UX-8 — Motion easing values realized in `prototype/tokens/motion.css`.** The easing curves
  in DESIGN.md/§8.6 were tagged `[ASSUMPTION]`; the prototype realized them. Source of record is now
  `prototype/tokens/motion.css`: `--crm-ease-standard: cubic-bezier(0.4,0,0.2,1)` (moves),
  `--crm-ease-decelerate: var(--iso-ease-out)` = `cubic-bezier(0.22,1,0.36,1)` (enters),
  `--crm-ease-accelerate: cubic-bezier(0.4,0,1,1)` (exits). **Durations are unchanged** (instant 0 ·
  fast 120 · base 200 · slow 320 — they match the spine). DESIGN.md easing block updated to these
  realized values and to point at `prototype/tokens/motion.css`.
- **DEC-UX-9 — Reduced-motion rule realized.** `motion.css` honors `prefers-reduced-motion: reduce`
  by collapsing the travel scalar `--crm-travel → 0` and flattening `base`/`slow` to `120ms` (fast
  stays `120ms`, instant stays `0ms`); opacity cross-fades survive and **all state changes + all
  feedback still fire** — only physical travel/snap-back is removed. The spine's `motion.reduced`
  prose (which said "all durations collapse to 0ms except opacity cross-fades") is reconciled to
  this realized behavior.
