# E1-S1 ‚Äî Thread TenantContext and enforce isolation

> **GitHub Epic ‚Üí Feature ‚Üí Story:** Epic **E1 ‚Äî Tenancy & Subsidiary** ‚Ä∫ Feature **1.1 ‚Äî Tenant Context & Isolation** ‚Ä∫ Story **E1-S1**
> **Cut:** Pilot ¬∑ **Depends on:** Epic 0 kernels (E0-S4 repository/4-beat, E0-S5 auth claims, E0-S6 permissions) ¬∑ *test-time:* E0-S11 (cross-tenant isolation E2E scaffold ‚Äî needed to run this story's isolation tests, not to build it)

## Story
As any user, I only ever see data in my tenant/subsidiary scope, so that one tenant's records can never leak into another's view or mutation.

## Implementation-Ready Acceptance Criteria
1. **AC1 ‚Äî Context from claims, consumed by repository (UC-5).** `TenantContext` is derived from the auth claims `{tenantId, subsidiaryId|null, roles[]}` and exposed via `app/providers.tsx`; the `LocalStorageRepository` resolves its scope **from this context only** ‚Äî never from a caller-supplied argument. Storage key is composed `crm:{tenantId}:{subsidiaryId|_parent}:{entity}` per the Repository contract.
   *Touches:* `src/app/providers.tsx`, `src/app/composition.ts`, `src/shared/data/LocalStorageRepository.ts`, `src/shared/auth/useAuth.ts` ¬∑ *Seam:* `Repository<T>` scope resolution ¬∑ *Schema/Map:* claims shape (ADR-009); n/a transition.
2. **AC2 ‚Äî No caller passes scope.** Every `list/get/create/update/remove` is scoped implicitly; the public `Repository<T>` signatures take **no** `tenantId`/`subsidiaryId` parameter, and an architecture-fitness assertion forbids any product-layer code from reading scope outside the context.
   *Touches:* `src/shared/data/Repository.ts`, `src/shared/data/LocalStorageRepository.ts` ¬∑ *Seam:* `Repository<T>` ¬∑ *Schema/Map:* n/a.
3. **AC3 ‚Äî Out-of-tenant access by ID ‚Üí 404 (UJ-1 edge case).** `get(id)`/`update`/`remove` of a record whose `tenantId` differs from the context resolves as **not-found (404)**, not 403 ‚Äî the record's existence is never disclosed (ADR-009 404-not-403; 4-beat authorize beat).
   *Touches:* `src/shared/data/LocalStorageRepository.ts`, `src/shared/auth/permissions.ts` ¬∑ *Seam:* 4-beat authorize ¬∑ *Schema/Map:* n/a.
4. **AC4 ‚Äî Scope resolution rule.** A `tenant_admin` with `subsidiaryId=null` sees the **whole tenant** (all subsidiary rows + parent-level rows); a subsidiary user (`subsidiaryId` set) sees **only their subsidiary + parent-level (`subsidiaryId=null`)** records. List queries filter accordingly; cross-subsidiary rows for a subsidiary user are invisible.
   *Touches:* `src/shared/data/LocalStorageRepository.ts`, `src/shared/auth/permissions.ts` ¬∑ *Seam:* `Repository<T>.list` scope filter ¬∑ *Schema/Map:* n/a.

## UX & Behavior (from prototype)
*Source: `prototype/app/store.jsx` (`setScope`), `prototype/app/tenancy.jsx` (`NotFoundView`), `prototype/app/app.jsx`. This story owns the two scope-edge surfaces every screen inherits.*

- **Scope re-query skeleton (felt isolation).** Changing scope (`setScope`) sets `scopeLoading: true`, then clears it after **~`--crm-base` + 220ms** ‚Äî so **every scoped screen shows its loading skeleton, then re-paints to the new scope's data**. The skeleton mirrors the real layout (never a spinner). This is the visible "the wall is real" beat (Principle 3); an empty scope resolves to the **empty** state, never "broken".
- **404 ‚Äî `NotFoundView` (cross-tenant never leaks existence).** Out-of-tenant/out-of-scope record access renders a **calm centered not-found**: `compass` glyph, eyebrow **"404 ¬∑ not found"**, heading **"Not found in this workspace"**, body explaining the record isn't part of `<scopeName>` (it may belong to another subsidiary/tenant), single primary action **"Back to <scopeName>"**. Copy is **explicitly not a permission warning** ‚Äî it never discloses that the record exists elsewhere. Also reached when a customer's originating-lead link points outside scope (`app.jsx`).
- **Motion:** scope re-query skeleton‚Üídata cross-fade at `--crm-base`/`decelerate`; reduced-motion keeps the timing + opacity fade, drops travel.
- *Visual oracle:* the locked/active scope chip appears in every shell screenshot; `NotFoundView` is reached via the E1-S5 roll-up "open a record from another workspace ‚Üó" link.

## Test Requirements (NFR-12)
- **Vitest:** repository scope-resolution unit tests ‚Äî admin (null) sees all tenant rows; subsidiary user sees own + parent-level only; cross-subsidiary rows filtered out; `get` of a foreign-`tenantId` id returns not-found; storage-key composition correct. Architecture-fitness test asserting no scope param crosses the public seam and product code never reads scope directly.
- **RTL:** `<QueryStateBoundary>`-wrapped scoped list shows empty state for an empty scope; data state for a populated scope.
- **Playwright:** **cross-tenant isolation** (`e2e/cross-tenant-isolation.spec.ts`) ‚Äî tenant A user cannot see or fetch-by-ID tenant B records (404); subsidiary user cannot see sibling-subsidiary rows. **architecture-fitness** (`e2e/architecture-fitness.spec.ts`) green.
