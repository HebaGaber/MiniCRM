# E1-S2 — Onboard a subsidiary

> **GitHub Epic → Feature → Story:** Epic **E1 — Tenancy & Subsidiary** › Feature **1.2 — Subsidiary Onboard / Offboard** › Story **E1-S2**
> **Cut:** Pilot · **Depends on:** E1-S1; Epic 0 kernels (E0-S7 events/audit, E0-S9 EntityForm, E0-S10 config precedence)

## Story
As a tenant admin, I add a subsidiary node, so that a child unit can operate in isolation within my tenant.

## Implementation-Ready Acceptance Criteria
1. **AC1 — tenant_admin-only (route + action guard).** The onboard route and the create action are gated by `can(actor, 'subsidiary', 'create', resource)` (ADR-015); non-admins are denied — out-of-tenant denial → 404, in-tenant denial → 403 audited as `Auth.RoleDenied`.
   *Touches:* `src/app/router.tsx`, `src/shared/auth/{guards.tsx,permissions.ts}`, `src/features/dashboard` (or tenancy feature surface) · *Seam:* two-gate authZ + 4-beat authorize · *Schema/Map:* n/a.
2. **AC2 — Inserts a Subsidiary row (data op, no infra).** Onboarding is a pure **data operation** through `Repository<Subsidiary>.create` — no provisioning/infra. Persists under the tenant scope; Zod-validated before persist.
   *Touches:* `src/shared/domain/{tenant.types.ts,schemas.ts}`, `src/shared/data/LocalStorageRepository.ts` · *Seam:* `Repository<T>.create` (4-beat mutate) · *Schema/Map:* `subsidiarySchema` (Zod).
3. **AC3 — Emits Tenant.SubsidiaryAdded + audit, one correlationId (UC-2).** The create's 4-beat emits one `Tenant.SubsidiaryAdded` `DomainEvent` and one immutable `AuditEvent` (before/after), both threaded with a single `correlationId` minted at action start.
   *Touches:* `src/shared/events/{bus.ts,auditLog.ts,eventTypes.ts,correlation.ts}` · *Seam:* 4-beat emit + audit · *Schema/Map:* `Tenant.SubsidiaryAdded` (canonical registry).
4. **AC4 — Inherits tenant config (precedence per E0-S10).** The new subsidiary starts with tenant-level config inherited; flag/config resolution follows most-specific-wins `subsidiary > tenant > system`, deny-wins (ADR-011), so an unset subsidiary value falls back to the tenant value.
   *Touches:* `src/shared/config/{FlagProvider.tsx,useConfig.ts}` · *Seam:* config precedence resolver · *Schema/Map:* n/a.
5. **AC5 — EntityForm four states (UC-1).** The onboard form uses the shared `EntityForm` template with all four states (idle/submitting/error/success) and RHF+Zod validation; optimistic create rolls back on error (fault-injection toggle).
   *Touches:* `src/shared/ui/templates/EntityForm`, `src/shared/ui/QueryStateBoundary.tsx` · *Seam:* optimistic mutation (`onMutate/onError/onSettled`) · *Schema/Map:* `subsidiarySchema` (Zod).

## Inherited Universal Conformance (subset)
- **UC-1 — Four UI states.** EntityForm renders all four states; create is optimistic with rollback.
- **UC-2 — Dual events, one correlationId.** `Tenant.SubsidiaryAdded` + audit share one correlationId.
- **UC-5 — Tenant scope.** The Subsidiary row is written under the admin's tenant scope only; cross-tenant create is impossible (scope from context).
- **TC — Traceability.** story → spec → code → test → `Closes #`; preview green; `sprint-status.yaml`; passes `bmad-code-review`.

## Test Requirements (NFR-12)
- **Vitest:** `subsidiarySchema` validation (valid/invalid); repository `create` writes under tenant scope; 4-beat emits exactly one `Tenant.SubsidiaryAdded` + one audit with a shared correlationId; permission cell — only `tenant_admin` may create; config inheritance resolves tenant value when subsidiary unset.
- **RTL:** EntityForm four states; optimistic create rollback on injected fault; non-admin sees no onboard affordance / route guard blocks.
- **Playwright:** onboard happy path (admin adds subsidiary → appears in switcher/list); **cross-tenant isolation** — created subsidiary is invisible to other tenants; non-admin cannot reach the onboard route.
