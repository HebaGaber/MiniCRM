# E1-S2 ‚ÄĒ Onboard a subsidiary

> **GitHub Epic ‚Üí Feature ‚Üí Story:** Epic **E1 ‚ÄĒ Tenancy & Subsidiary** ‚Äļ Feature **1.2 ‚ÄĒ Subsidiary Onboard / Offboard** ‚Äļ Story **E1-S2**
> **Cut:** Pilot ¬∑ **Depends on:** E1-S1; Epic 0 kernels (E0-S7 events/audit, E0-S9 EntityForm, E0-S10 config precedence)

## Story
As a tenant admin, I add a subsidiary node, so that a child unit can operate in isolation within my tenant.

## Implementation-Ready Acceptance Criteria
1. **AC1 ‚ÄĒ tenant_admin-only (route + action guard).** The onboard route and the create action are gated by `can(actor, 'subsidiary', 'create', resource)` (ADR-015); non-admins are denied ‚ÄĒ out-of-tenant denial ‚Üí 404, in-tenant denial ‚Üí 403 audited as `Auth.RoleDenied`.
   *Touches:* `src/app/router.tsx`, `src/shared/auth/{guards.tsx,permissions.ts}`, `src/features/dashboard` (or tenancy feature surface) ¬∑ *Seam:* two-gate authZ + 4-beat authorize ¬∑ *Schema/Map:* n/a.
2. **AC2 ‚ÄĒ Inserts a Subsidiary row (data op, no infra).** Onboarding is a pure **data operation** through `Repository<Subsidiary>.create` ‚ÄĒ no provisioning/infra. Persists under the tenant scope; Zod-validated before persist.
   *Touches:* `src/shared/domain/{tenant.types.ts,schemas.ts}`, `src/shared/data/LocalStorageRepository.ts` ¬∑ *Seam:* `Repository<T>.create` (4-beat mutate) ¬∑ *Schema/Map:* `subsidiarySchema` (Zod).
3. **AC3 ‚ÄĒ Emits Tenant.SubsidiaryAdded + audit, one correlationId (UC-2).** The create's 4-beat emits one `Tenant.SubsidiaryAdded` `DomainEvent` and one immutable `AuditEvent` (before/after), both threaded with a single `correlationId` minted at action start.
   *Touches:* `src/shared/events/{bus.ts,auditLog.ts,eventTypes.ts,correlation.ts}` ¬∑ *Seam:* 4-beat emit + audit ¬∑ *Schema/Map:* `Tenant.SubsidiaryAdded` (canonical registry).
4. **AC4 ‚ÄĒ Inherits tenant config (precedence per E0-S10).** The new subsidiary starts with tenant-level config inherited; flag/config resolution follows most-specific-wins `subsidiary > tenant > system`, deny-wins (ADR-011), so an unset subsidiary value falls back to the tenant value.
   *Touches:* `src/shared/config/{FlagProvider.tsx,useConfig.ts}` ¬∑ *Seam:* config precedence resolver ¬∑ *Schema/Map:* n/a.
5. **AC5 ‚ÄĒ EntityForm four states (UC-1).** The onboard form uses the shared `EntityForm` template with all four states (idle/submitting/error/success) and RHF+Zod validation; optimistic create rolls back on error (fault-injection toggle).
   *Touches:* `src/shared/ui/templates/EntityForm`, `src/shared/ui/QueryStateBoundary.tsx` ¬∑ *Seam:* optimistic mutation (`onMutate/onError/onSettled`) ¬∑ *Schema/Map:* `subsidiarySchema` (Zod).

## UX & Behavior (from prototype)
*Source: `prototype/app/tenancy.jsx` (`SubsidiariesPage`, `OnboardForm`). Build identical to the prototype.*

- **Subsidiaries list (admin only)** ‚ÄĒ columns **Name** (icon + name; offboarded rows render at **0.55 opacity** with an **"Offboarded"** pill) ¬∑ **Parent** (parent name or "Top-level") ¬∑ **Region** ¬∑ **Created** (right-aligned). Toolbar: a (non-wired) search + an **"Include offboarded"** toggle that reveals soft-deleted rows. Row actions: active ‚Üí **"Offboard subsidiary"** (`log-out`, danger); offboarded ‚Üí **"Offboarded ‚ÄĒ read only"** (`lock`, inert). Primary action **"Onboard subsidiary"**.
- **States:** loading (DataTable skeleton) ¬∑ **empty** (`network` icon, **"No subsidiaries yet"**, primary action auto-focused ‚ÄĒ only when no active subs *and* "include offboarded" off) ¬∑ error (retry) ¬∑ ready.
- **`OnboardForm` modal ‚ÄĒ fields in order:** **Subsidiary name** (required) ¬∑ **Parent** (select: **"Top-level (no parent)"** + each active sub). Info strip: *"inherits the tenant configuration by default."* Focus-trap, first-field autofocus, `Esc` cancels.
- **Optimistic create + rollback:** row appears immediately. **OutcomePicker** ("Simulate server response: Success / Server error"). On **Server error** the optimistic row is removed after ~700ms (snap-back at `--crm-slow`) and a **persistent danger toast** fires ‚ÄĒ **"Couldn't onboard ‚ÄĒ rolled back"** ‚ÄĒ with a **Retry** action.
- **Motion:** modal enter `crm-pop` at `--crm-base`; row insert/optimistic apply `--crm-instant`; rollback snap-back `--crm-slow`; toast enter `--crm-fast`.

### Correct-course fidelity pins ‚ÄĒ UI source of truth (DEC-CC-5, 2026-06-08)
*Prototype `prototype/app/tenancy.jsx` (`SubsidiariesPage` L11‚Äď66, `OnboardForm` L80‚Äď137) is the oracle; these pin the audit findings so the dev-story re-run matches it exactly. Tightened restatements, not new scope.*

- **Onboard info-strip:** background token **`--iso-blue-3-50`** (not `--iso-brand-soft`/`--iso-blue-3-100`).
- **OutcomePicker:** the bordered *"Simulate server response"* strip with the **`flask-conical`** lead icon, **dashed `--iso-blue-3-300`** border, and a segmented Success/Server-error toggle (not two plain buttons).
- **Rollback toast title:** exact string **"Couldn't onboard ‚ÄĒ rolled back."** (trailing period); body *"The subsidiary was removed. Try again."*; **persistent** (no auto-dismiss) with a **Retry** action.
- **Modal scrim (DEC-CC-4 / ¬ß8.6):** the **DS scrim + blur tokens** (`--crm-scrim` + `--crm-backdrop-blur`), **not** a raw `rgba()` ‚ÄĒ neither the prototype's `rgba(15,22,38,0.42)` nor the impl's `rgba(0,0,0,0.25)` is to be inlined. Modal is **top-anchored (~64px)**, not vertically centered.
- **Toggle reduced-motion (¬ß8.6):** the knob travel uses the **`--crm-travel`** scalar (`translateX(calc(var(--crm-travel) * 16px))`), so reduced-motion drops the slide ‚ÄĒ not a hardcoded `translateX(16px)`.
