# E1-S5 ‚Äî Cross-subsidiary roll-up (read model)

> **GitHub Epic ‚Üí Feature ‚Üí Story:** Epic **E1 ‚Äî Tenancy & Subsidiary** ‚Ä∫ Feature **1.4 ‚Äî Parent Roll-up Read Model** ‚Ä∫ Story **E1-S5**
> **Cut:** Pilot ¬∑ **Depends on:** E1-S1 (scope seam), E1-S2 (subsidiaries to aggregate)

## Story
As a tenant admin, I see aggregate counts across my subsidiaries, so that I have a parent-level view of the whole tenant.

## Implementation-Ready Acceptance Criteria
1. **AC1 ‚Äî Gated by `rollup.view`; admin tenant-wide, others scoped (DEC-CC-6).** The roll-up surface and its aggregate query are gated by `can(actor, 'rollup', 'view', resource)` (ADR-015). A **`tenant_admin`** (`subsidiaryId=null`) sees **tenant-wide** aggregates; **`sales`/`support`/`viewer`** are **not** denied ‚Äî they see a **scope-limited** roll-up (own subsidiary + parent-level) per AC3 and the ¬ß6.2 roll-up row. *(Corrects the prior "tenant_admin-only" wording, which contradicted AC3; ruled conflict **C-5 ‚Üí keep spec**: the prototype's admin-only nav gate is the stale artifact, not the access model.)*
   *Touches:* `src/app/router.tsx`, `src/shared/auth/{guards.tsx,permissions.ts}`, `src/features/dashboard` ¬∑ *Seam:* two-gate authZ ¬∑ *Schema/Map:* `rollup.view` cell (admin `allow`, others `view`).
2. **AC2 ‚Äî Parent-level read model aggregating subsidiary data; never a cross-boundary write.** The roll-up is a **read-only** aggregation (counts per subsidiary / per entity) computed over the admin's tenant scope. It is strictly a read model ‚Äî **no cross-boundary writes** (Glossary: Roll-up); it issues no mutations and emits no domain events.
   *Touches:* `src/features/dashboard/widgets`, `src/shared/data/LocalStorageRepository.ts` (read-only `list`/count) ¬∑ *Seam:* `Repository<T>.list` aggregation (read-only) ¬∑ *Schema/Map:* n/a.
3. **AC3 ‚Äî Subsidiary user sees only their own scope (UJ-5 edge case).** A subsidiary user opening the same surface sees aggregates for **their own scope only** (own subsidiary + parent-level), never sibling-subsidiary data ‚Äî the read model re-uses the E1-S1 scope filter, so it cannot disclose cross-subsidiary counts.
   *Touches:* `src/shared/data/LocalStorageRepository.ts`, `src/shared/auth/permissions.ts`, `src/features/dashboard` ¬∑ *Seam:* repository scope filter ¬∑ *Schema/Map:* n/a.
4. **AC4 ‚Äî Four states (UC-1); empty fires on a zero grand total.** The roll-up dashboard renders loading/empty/error/data via `<QueryStateBoundary>`; the **empty** state fires when **`grandTotal === 0`** ‚Äî **regardless of subsidiary count**. A tenant that has subsidiaries but no records (all-zero rows) shows the **empty** state, *not* an all-zeros ready table and *not* an error. (The impl currently also requires `visibleSubs.length === 0`, which wrongly shows an all-zeros table ‚Äî pin the predicate to `grandTotal === 0` alone.)
   *Touches:* `src/shared/ui/templates/Dashboard`, `src/shared/ui/QueryStateBoundary.tsx`, `src/features/dashboard/widgets`, `src/features/dashboard/useRollup.ts` ¬∑ *Seam:* four-state boundary ¬∑ *Schema/Map:* n/a.

## UX & Behavior (from prototype)
*Source: `prototype/app/tenancy.jsx` (`RollupPage`). Build identical to the prototype. NB: the per-subsidiary roll-up is THIS surface ‚Äî distinct from the E5-S2 dashboard conversion funnel (DEC-UX-7).*

- **Read-only aggregate table** ‚Äî columns **Subsidiary ¬∑ Leads ¬∑ Customers ¬∑ Tickets ¬∑ Total** (tabular numerals). A **"Tenant total"** footer row sums everything; the grand total renders in **brand color**. A **"Read-only"** lock pill sits in the header.
- **Scope behavior:** in **tenant** scope it lists every active subsidiary **plus a "Parent level (shared)" row**; in a **single-subsidiary** scope it shows only that sub + parent, with copy *"Sibling subsidiaries are not shown."*
- **States:** loading (row skeletons) ¬∑ empty (**"Nothing to roll up yet"**) ¬∑ error (**"Can't load roll-up"** + retry) ¬∑ ready.
- **Edge-case hook:** a footer link **"open a record from another workspace ‚Üó"** deliberately routes to the **404 `NotFoundView`** (E1-S1) ‚Äî exercising the calm cross-tenant not-found.
- **Motion:** rows skeleton‚Üíready at `--crm-base`; re-query on scope change via `scopeLoading`.

## Test Requirements (NFR-12)
- **Vitest:** aggregation correctness ‚Äî admin counts span all tenant subsidiaries + parent-level; subsidiary user counts cover only own + parent-level; read model issues no writes/events (assert no mutation path invoked); permission cells ‚Äî `view` granted to tenant_admin, scoped for subsidiary user, denied where applicable.
- **RTL:** Dashboard four states (loading/empty/error/data); empty-tenant shows empty state; widgets render counts for the resolved scope.
- **Playwright:** admin roll-up shows tenant-wide aggregates; **cross-tenant isolation** ‚Äî roll-up never includes another tenant's counts; **UJ-5 edge case** ‚Äî subsidiary user on the same surface sees only their own scope's aggregates, no sibling-subsidiary numbers.
