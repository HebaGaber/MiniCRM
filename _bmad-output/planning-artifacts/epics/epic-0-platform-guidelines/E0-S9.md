# E0-S9 — Build the shared component inventory + page templates

> **GitHub Epic → Feature → Story:** Epic **E0 — Platform Guidelines & Standards** › Feature **0.5 — UI Kernel & Component Inventory** › Story **E0-S9**
> **Cut:** Pilot · **Depends on:** E0-S1 (STATUS_TONE), E0-S3 (Page<T> for DataTable)

## Story
As the platform, all screens are assembled from one fixed component inventory and page templates wired to a four-state boundary and design tokens, so that every feature looks and behaves consistently and no view hardcodes styling or status color.

## Implementation-Ready Acceptance Criteria
1. **AC1 — Component inventory.** Build `AppShell` (nav + tenant/subsidiary switcher), `DataTable`, `StatusPill` (tone **only** from `STATUS_TONE`), `Toolbar`, `FilterBar`, `EntityForm` fields, `ConfirmDialog`, `Toast`, `EmptyState`, `ErrorState`, `Skeleton`, and **`RecordPager`** (DEC-CC-3 — prev/next + Side/Full view, see the UX block).
   *Touches:* `src/shared/ui/components/*` · *Seam:* UI kernel · *Schema/Map:* `STATUS_TONE` (E0-S1) for `StatusPill`.
2. **AC2 — Page templates.** Build `ListPage`, `DetailPage`, `EntityForm`, and `Dashboard` templates that features specialize.
   *Touches:* `src/shared/ui/templates/{ListPage,DetailPage,EntityForm,Dashboard}` · *Seam:* UI kernel · *Schema/Map:* `Page<T>`/`ListQuery` (E0-S3) for list templates.
3. **AC3 — Four-state harness (UC-1).** Ship `QueryStateBoundary` rendering loading/empty/error/data states; any data view plugs into it (consumes `Skeleton`/`EmptyState`/`ErrorState`).
   *Touches:* `src/shared/ui/QueryStateBoundary.tsx` · *Seam:* four-state boundary · *Schema/Map:* n/a.
4. **AC4 — Tokens only (NFR-10).** No hardcoded hex/px/font anywhere; styling reads design tokens.
   *Touches:* `src/shared/ui/tokens.css`, `src/shared/ui/components/*`, `src/shared/ui/templates/*` · *Seam:* UI kernel · *Schema/Map:* design tokens.
5. **AC5 — Destructive/convert require ConfirmDialog.** Destructive and convert actions are gated by `ConfirmDialog`.
   *Touches:* `src/shared/ui/components/ConfirmDialog`, `src/shared/ui/templates/*` · *Seam:* UI kernel · *Schema/Map:* n/a.
6. **AC6 — Accessibility & responsiveness.** Components are keyboard-accessible and responsive down to tablet width.
   *Touches:* `src/shared/ui/components/*`, `src/shared/ui/templates/*` · *Seam:* UI kernel · *Schema/Map:* n/a.

## UX & Behavior (from prototype)
*Source: `prototype/app/{shell,components_core,components_data,templates,shared_records}.jsx`; tokens `prototype/tokens/{motion.css,colors_and_type.css}`. Build the shared inventory to behave identically to the prototype. Visual oracle: every shell screenshot, plus `prototype/screenshots/01-roles.png` (component gallery), `sideview.png`.*

- **AppShell** — CSS grid `[logo | topbar]` (64px) over `[sidebar | main]`; app background `--iso-blue-3-50`. **Sidebar** animates `248px ↔ 64px` (collapsed) via `grid-template-columns` at `--crm-base`/`--crm-ease-standard`; role-gated grouped nav (**Workspace**, **Tenancy**, pinned-bottom **Build** with a "Components" gallery entry); active item = `--iso-brand` bg + white text; nav color transitions at `--crm-fast`; collapse toggle = hamburger by the wordmark. **Topbar** (left→right): **ScopeSwitcher** (E1-S4) · **Search box** (placeholder *"Search records, tickets, people…"* + `⌘K` hint chip — **decorative/non-functional** in pilot, focus toggles ring only) · **NotificationsBell** (E0-S12/E5-S3) · **UserMenu** (avatar + name + role; dropdown: email, **"Switch role (demo)"**, "View all notifications", "Sign out").
- **StatusPill** — tone comes **only** from `STATUS_TONE` (E0-S1) → the five tone tokens (`neutral·info·success·warning·danger`); pill **always carries a text label** (status never by color alone) plus a dot or icon; uniform shape/size, **only the tone token differs**. Tone change is the pill's **only** animation — cross-fade at `--crm-fast`. Gallery shows DOT and ICON variants (`01-roles.png`).
- **DataTable** — renders all four states internally; rows keyboard-operable (`tabIndex`, Enter/Space open the row), sortable headers with `aria-sort`, active-row highlight = brand-soft fill + inset brand bar; column sort toggles asc/desc.
- **Toast / ToastHost** — bottom-center stack, **max 3** (older collapse); **success auto-dismisses ~4s**, **danger/error persist** (manually dismissible, `×`); optional action button (e.g. "Retry"); tone icons success `check-circle` / danger `rotate-ccw` / warning `alert-triangle`; enter at `--crm-fast` `decelerate`, exit `--crm-fast` `accelerate`; `aria-live` polite (success) / assertive (error). Toasts are **own-action outcomes, not a feed**.
- **ModalShell / ConfirmDialog** — overlay = the **DS scrim token** (`--iso-overlay-scrim`) + **DS blur token** — **not a raw `rgba()`/`px`** (NFR-10/§8.4: no hardcoded hex/px). The prototype's realized literal (`rgba(15,22,38,0.42)` + `blur(2px)`) is logged as the realized value in `_bmad-output/decision-log.md` (**DEC-CC-4**); add the scrim/blur to the token layer if the DS does not already expose them. Panel enters with `crm-pop` at `--crm-base`; **focus-trapped**, `Esc` cancels, click-outside cancels; `ConfirmDialog` focuses the **safe** control by default, destructive confirm is `danger`-toned and **never** default-focused. Required for delete + convert (**convert hands off to the saga inspector — no double-confirm**).
- **EmptyState** — brand-soft icon + title + optional **scope line** + body + (optional, **auto-focused**) primary action; no disabled dead button if the role can't perform it. **ErrorState** — contained panel (`cloud-off` glyph) + **Retry**, *not* a toast/takeover. **Skeleton** (`crm-skel`) — mirrors the real layout, **never a bare spinner**.
- **Four-state contract** (`QueryStateBoundary`): loading → empty → error → ready; skeleton→ready cross-fades at `--crm-base`/`decelerate`; **mutation failures use the toast/rollback path, not ErrorState**.
- **Elevation & z-index** (consumed from DS): elevation reserved for transient overlays only (dropdown/modal/toast); the **saga inspector is in-page and does NOT elevate**. Z-index ladder: dropdown 100 · sticky 200 · overlay 800 · modal 900 · toast 1000.
- **Motion tokens** are the realized `--crm-*` set in `prototype/tokens/motion.css` (DEC-UX-8/9): `instant 0` · `fast 120` · `base 200` · `slow 320`; easing `--crm-ease-standard/-decelerate/-accelerate`; `--crm-travel` scalar honoring `prefers-reduced-motion`. **No raw hex/px/font/ms/curve in components** (NFR-10/§8.4) — reference token names only.
- **✅ C-4 RESOLVED (DEC-CC-3) — `RecordPager` + Side-view ARE in the inventory; build them here.** Build a **`RecordPager`** sticky detail bar: prev/next chevrons (respecting the **active list filter/sort order**, ids carried from the list), **"N of M · <noun>"**, a **Side/Full view toggle**, Close `×`; keyboard `↑/←/k`=prev, `↓/→/j`=next, `Esc`=close (**suppressed while focus is in an input/textarea/select**, and only active when a record is open and no modal). Build the **Side view** = the list rendered beside the detail panel (open row highlighted), **persisted per-user** in `localStorage('crm-view-mode')`. Both are hosted on the `DetailPage` template — **no other** new layouts (NFR-10 exception logged in `_bmad-output/decision-log.md`). Source: `prototype/app/{app,shared_records}.jsx`; oracle `prototype/screenshots/sideview.png` ("2 of 4 · Leads").
- **Component gallery** (`gallery.jsx`, sidebar "Components") is a dev-only inventory page; not a shipping product surface.
