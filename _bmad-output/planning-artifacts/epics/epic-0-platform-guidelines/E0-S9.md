# E0-S9 — Build the shared component inventory + page templates

> **GitHub Epic → Feature → Story:** Epic **E0 — Platform Guidelines & Standards** › Feature **0.5 — UI Kernel & Component Inventory** › Story **E0-S9**
> **Cut:** Pilot · **Depends on:** E0-S1 (STATUS_TONE), E0-S3 (Page<T> for DataTable)

## Story
As the platform, all screens are assembled from one fixed component inventory and page templates wired to a four-state boundary and design tokens, so that every feature looks and behaves consistently and no view hardcodes styling or status color.

## Implementation-Ready Acceptance Criteria
1. **AC1 — Component inventory.** Build `AppShell` (nav + tenant/subsidiary switcher), `DataTable`, `StatusPill` (tone **only** from `STATUS_TONE`), `Toolbar`, `FilterBar`, `EntityForm` fields, `ConfirmDialog`, `Toast`, `EmptyState`, `ErrorState`, `Skeleton`.
   *Touches:* `src/shared/ui/components/*` · *Seam:* UI kernel · *Schema/Map:* `STATUS_TONE` (E0-S1) for `StatusPill`.
2. **AC2 — Page templates.** Build `ListPage`, `DetailPage`, `EntityForm`, and `Dashboard` templates that features specialize.
   *Touches:* `src/shared/ui/templates/{ListPage,DetailPage,EntityForm,Dashboard}` · *Seam:* UI kernel · *Schema/Map:* `Page<T>`/`ListQuery` (E0-S3) for list templates.
3. **AC3 — Four-state harness (UC-1).** Ship `QueryStateBoundary` rendering loading/empty/error/data states; any data view plugs into it (consumes `Skeleton`/`EmptyState`/`ErrorState`).
   *Touches:* `src/shared/ui/QueryStateBoundary.tsx` · *Seam:* four-state boundary · *Schema/Map:* n/a.
4. **AC4 — Tokens only (NFR-10).** No hardcoded hex/px/font anywhere; styling reads design tokens.
   *Touches:* `src/shared/ui/tokens.css`, `src/shared/ui/components/*`, `src/shared/ui/templates/*` · *Seam:* UI kernel · *Schema/Map:* design tokens.
5. **AC5 — Destructive/convert require ConfirmDialog.** Destructive and convert actions are gated by `ConfirmDialog`.
   *Touches:* `src/shared/ui/components/ConfirmDialog`, `src/shared/ui/templates/*` · *Seam:* UI kernel · *Schema/Map:* n/a.
6. **AC6 — Accessibility & responsiveness.** Components are keyboard-accessible and responsive down to tablet width.
   *Touches:* `src/shared/ui/components/*`, `src/shared/ui/templates/*` · *Seam:* UI kernel · *Schema/Map:* n/a.

## Inherited Universal Conformance (subset)
- **UC-1 — Four UI states.** `QueryStateBoundary` is the harness every data-backed view in Epics 1–5 plugs into.
