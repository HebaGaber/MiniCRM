# E0-S12 ÔÇö Notifications kernel

> **GitHub Epic Ôćĺ Feature Ôćĺ Story:** Epic **E0 ÔÇö Platform Guidelines & Standards** ÔÇ║ Feature **0.7 ÔÇö Notifications Kernel** ÔÇ║ Story **E0-S12**
> **Cut:** Pilot ┬Ě **Depends on:** E0-S7 (domain-event bus), E0-S9 (four-state surface)

## Story
As the platform, in-app notifications are a shared-layer capability driven off the domain-event bus ÔÇö not feature-owned ÔÇö so that any event can produce a scoped, recipient-targeted notification without bespoke writes.

## Implementation-Ready Acceptance Criteria
1. **AC1 ÔÇö Service + hook.** `src/shared/notifications` ships a `NotificationService` that **subscribes to the `DomainEvent` bus** (E0-S7) and a `useNotifications()` hook for consumers.
   *Touches:* `src/shared/notifications/NotificationService.ts`, `src/shared/notifications/useNotifications.ts` ┬Ě *Seam:* notifications kernel subscribing to event bus ┬Ě *Schema/Map:* `DomainEvent` (E0-S7).
2. **AC2 ÔÇö Notification data shape.** A notification carries `{ recipient, type, payload, read/unread, tenantId, subsidiaryId }` ÔÇö scoped to the recipient's tenant/subsidiary.
   *Touches:* `src/shared/notifications/NotificationService.ts` ┬Ě *Seam:* notifications kernel ┬Ě *Schema/Map:* notification shape.
3. **AC3 ÔÇö Recipient-mapping handlers (UC-5).** Handlers map event type Ôćĺ recipient (e.g. `Ticket.Assigned Ôćĺ assigneeId`; `Lead.Converted Ôćĺ lead ownerId`), scoped to the recipient's tenant/subsidiary. Notifications are driven off domain events **only ÔÇö never bespoke writes**.
   *Touches:* `src/shared/notifications/NotificationService.ts` ┬Ě *Seam:* notifications kernel + event bus ┬Ě *Schema/Map:* event-type Ôćĺ recipient map.
4. **AC4 ÔÇö Four-state surface scaffold (UC-1).** A four-state notifications surface scaffold (loading/empty/error/data via `QueryStateBoundary`) is provided, consumed later by E5-S3 / E4-S3.
   *Touches:* `src/shared/notifications/useNotifications.ts` (+ `src/shared/ui/QueryStateBoundary.tsx`) ┬Ě *Seam:* four-state boundary ┬Ě *Schema/Map:* n/a.

- **UC-5 ÔÇö Tenant scope.** Recipient mapping is scoped to the recipient's tenant/subsidiary; no cross-tenant notification leaks.
