# E0-S6 — Route guard + action guard from the permission matrix (ADR-015 predicates)

> **GitHub Epic → Feature → Story:** Epic **E0 — Platform Guidelines & Standards** › Feature **0.3 — Auth & RBAC Kernel** › Story **E0-S6**
> **Cut:** Pilot · **Depends on:** E0-S5 (claims), E0-S7 (audit bus for `Auth.RoleDenied`)

## Story
As the platform, two authorization gates (screen access and per-mutation access) decide every grant from one data-encoded permission matrix with predicate semantics, so that no feature invents its own access logic and deny always wins.

## Implementation-Ready Acceptance Criteria
1. **AC1 — Two gates.** A **route guard** decides screen access; an **action guard** decides per-mutation access on a specific record within the caller's tenant/subsidiary. Both delegate to one `can(actor, capability, action, resource)` predicate.
   *Touches:* `src/shared/auth/guards.tsx`, `src/shared/auth/permissions.ts` · *Seam:* authZ guards · *Schema/Map:* Grant table.
2. **AC2 — Deny-wins, default deny.** Conflicting or absent grants resolve to deny; the default grant for any unlisted `(capability, role)` cell is `deny`.
   *Touches:* `src/shared/auth/permissions.ts` · *Seam:* `can()` predicate · *Schema/Map:* Grant table.
3. **AC3 — §2.2 matrix encoded as data, cell-tested.** The §2.2 permission matrix is encoded as a `Grant` table (`Capability × Role → Grant`) and unit-tested cell-by-cell.
   *Touches:* `src/shared/auth/permissions.ts`, `src/shared/auth/permissions.test.ts` · *Seam:* `can()` predicate · *Schema/Map:* Grant table.
4. **AC4 — Out-of-tenant → 404 (UC-5).** Access to a record outside the caller's tenant returns `404` (existence is never disclosed), not `403`.
   *Touches:* `src/shared/auth/guards.tsx`, `src/shared/auth/permissions.ts` · *Seam:* authZ guards · *Schema/Map:* n/a.
5. **AC5 — Denied action audited (UC-2).** A denied action emits `Auth.RoleDenied` to the dual streams with a `correlationId`.
   *Touches:* `src/shared/auth/permissions.ts` (+ `shared/events/auditLog.ts`, `shared/events/bus.ts`) · *Seam:* `can()` + event/audit bus · *Schema/Map:* `Auth.RoleDenied`.
6. **AC6 — ADR-015 predicate model (DELTA).** Encode predicates: `Grant ∈ allow|deny|view|own|restricted`. `own` ⇒ `isOwned(actor, resource)` = (`ownerId|assigneeId|actorId === userId`). `restricted` ⇒ `isOwned AND action ∈ {softDelete, export}`. **Hard-delete is NEVER granted** (tenant_admin only). Concretely: "View audit/events" → `own` (sales/support), `allow` (tenant_admin), `deny` (viewer); "Delete(soft)/export" → `restricted` (sales/support).
   *Touches:* `src/shared/auth/permissions.ts`, `src/shared/auth/permissions.test.ts` · *Seam:* `can()` predicate · *Schema/Map:* Grant table + predicates.

## Inherited Universal Conformance (subset)
- **UC-2 — Dual events, one correlationId.** Denials emit `Auth.RoleDenied` on both streams.
- **UC-5 — Tenant scope, out-of-tenant → 404.** Cross-tenant access disclosed as `404`.
- **TC — Traceability.** story → spec → code → test → `Closes #`; preview green; `sprint-status.yaml`; passes `bmad-code-review`.

## Test Requirements (NFR-12)
- **Vitest:** the cell-by-cell matrix test enumerates **every** `(capability, role)` cell against fixtures for owned/not-owned, read/mutate, and soft/hard-delete; asserts `own` requires ownership, `restricted` allows only `{softDelete, export}` on owned records, hard-delete is denied for all roles incl. sales/support, default cells deny, and the concrete §2.2 rows ("View audit/events", "Delete(soft)/export") resolve as specified.
- **RTL:** route guard renders/blocks a screen by role; action guard disables/permits a mutation control on owned vs. not-owned records.
- **Playwright:** denial + out-of-tenant 404 paths exercised by the cross-tenant isolation E2E (E0-S11).
