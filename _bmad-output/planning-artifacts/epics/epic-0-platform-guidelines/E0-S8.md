# E0-S8 — Structured logger

> **GitHub Epic → Feature → Story:** Epic **E0 — Platform Guidelines & Standards** › Feature **0.4 — Event, Audit & Logging Kernel** › Story **E0-S8**
> **Cut:** Pilot · **Depends on:** E0-S7 (correlationId + shared redaction)

## Story
As the platform, all logging emits structured JSON with tenant/actor context and a `correlationId`, with PII masked, so that operations are traceable end-to-end without leaking sensitive data.

## Implementation-Ready Acceptance Criteria
1. **AC1 — Structured JSON shape.** The logger emits JSON `{ ts, level, tenantId, subsidiaryId, actorId, msg, correlationId, ... }`; context (tenant/subsidiary/actor/correlationId) is threaded from the auth context and the active operation.
   *Touches:* `src/shared/events/logger.ts` (+ `shared/events/correlation.ts`) · *Seam:* logger · *Schema/Map:* log record shape.
2. **AC2 — PII masking (NFR-8).** Emails and phones are masked; no tokens, passwords, or PII bodies are ever logged. Masking reuses the E0-S7 redaction rules.
   *Touches:* `src/shared/events/logger.ts` · *Seam:* logger · *Schema/Map:* masking/redaction rules.
3. **AC3 — Levels.** Supports `error`, `warn`, `info`, `debug` levels.
   *Touches:* `src/shared/events/logger.ts` · *Seam:* logger · *Schema/Map:* n/a.
