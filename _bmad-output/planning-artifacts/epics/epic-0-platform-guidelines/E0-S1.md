# E0-S1 — Author status enums, transition maps, and tone

> **GitHub Epic → Feature → Story:** Epic **E0 — Platform Guidelines & Standards** › Feature **0.1 — Domain & Status Kernel** › Story **E0-S1**
> **Cut:** Pilot · **Depends on:** none (foundational kernel)

## Story
As the platform, all statuses and legal transitions are defined once, so that no feature hardcodes a status or invents a transition.

## Implementation-Ready Acceptance Criteria
1. **AC1 — Status literals.** Define `LeadStatus`, `CustomerStatus` (5-state per DEC-1: `prospect|onboarding|active|inactive|churned`), `TicketStatus`, `TicketPriority`, `LeadSource`, `Role`, `TenantStatus` as string-literal unions (no enums-as-classes; literal unions only).
   *Touches:* `src/shared/domain/status.ts` · *Seam:* domain kernel · *Schema/Map:* literal unions.
2. **AC2 — Transition maps.** Author `LEAD_TRANSITIONS`, `CUSTOMER_TRANSITIONS` (newly authored, DEC-1), and `TICKET_TRANSITIONS` (Flag C). Lead: `new→[contacted,disqualified]`, `contacted→[qualified,disqualified]`, `qualified→[converted,disqualified]`, `disqualified→[contacted]`, `converted→[]`. Customer: `prospect→[onboarding]`, `onboarding→[active]`, `active→[inactive,churned]`, `inactive→[active,churned]`, `churned→[]`. Ticket: `open→[in_progress,pending,closed]`, `in_progress→[pending,resolved,open]`, `pending→[in_progress,resolved]`, `resolved→[closed,open]`, `closed→[]` (terminal — **reopen only `resolved→open`**).
   *Touches:* `src/shared/domain/status.ts` · *Seam:* transition maps · *Schema/Map:* `LEAD_TRANSITIONS` / `CUSTOMER_TRANSITIONS` / `TICKET_TRANSITIONS`.
3. **AC3 — Status tone.** `STATUS_TONE` maps every status value to a semantic tone token; it MUST include `customer.prospect` and `customer.onboarding`. Tone is the only source `StatusPill` (E0-S9) may read for status color.
   *Touches:* `src/shared/domain/status.ts` · *Seam:* domain kernel · *Schema/Map:* `STATUS_TONE`.
4. **AC4 — `canTransition` helper.** Export `canTransition(entity, from, to): boolean` that resolves the correct map by entity kind and returns whether `to` is in the legal set for `from`. This is the single helper the service/repository layer calls before any state move.
   *Touches:* `src/shared/domain/status.ts` · *Seam:* domain kernel helper consumed by 4-beat mutate beat · *Schema/Map:* all three transition maps.
5. **AC5 — Illegal transition → 422.** Document and expose the contract that an illegal transition surfaces as `422` (UC-3) when invoked via `POST /{resource}/{id}/transition`; state never changes via `PATCH status`. (Enforcement lands in E0-S4; this story guarantees `canTransition` returns `false` for the illegal case.)
   *Touches:* `src/shared/domain/status.ts` · *Seam:* transition contract · *Schema/Map:* transition maps.
6. **AC6 — Exhaustive transition tests.** Unit tests cover **every** legal transition in each map and a representative sample of illegal transitions per map (incl. terminal states `converted`/`churned`/`closed` rejecting all moves, and `resolved→open` being the only legal reopen).
   *Touches:* `src/shared/domain/status.test.ts` · *Seam:* domain kernel · *Schema/Map:* all three maps.
