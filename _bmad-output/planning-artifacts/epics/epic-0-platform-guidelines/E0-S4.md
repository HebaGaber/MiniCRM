# E0-S4 ‚Äî Implement LocalStorageRepository honoring the 4-beat + REST contract

> **GitHub Epic ‚Üí Feature ‚Üí Story:** Epic **E0 ‚Äî Platform Guidelines & Standards** ‚Ä∫ Feature **0.2 ‚Äî Repository Seam & Data Kernel** ‚Ä∫ Story **E0-S4**
> **Cut:** Pilot ¬∑ **Depends on:** E0-S3 (interface), E0-S5/E0-S6 (auth + guards for the authorize beat), E0-S7 (event/audit bus for the emit/audit beats)

## Story
As the platform, the pilot persists through one localStorage adapter that runs every mutation through the 4-beat use case and returns REST-shaped envelopes, so that the production frontend behaves identically when the adapter swaps to a server.

## Implementation-Ready Acceptance Criteria
1. **AC1 ‚Äî Key scheme.** Persist under `crm:{tenantId}:{subsidiaryId|_parent}:{entity}` (parent-level rows use the `_parent` segment).
   *Touches:* `src/shared/data/LocalStorageRepository.ts` ¬∑ *Seam:* `Repository<T>` impl ¬∑ *Schema/Map:* key scheme.
2. **AC2 ‚Äî Scope from auth context, never from caller (UC-5).** `tenantId`/`subsidiaryId` are read **inside** the repository from the auth context (E0-S5); callers cannot pass scope. `tenant_admin` (`subsidiaryId=null`) sees all tenant rows; a subsidiary user sees own + null rows.
   *Touches:* `src/shared/data/LocalStorageRepository.ts` ¬∑ *Seam:* `Repository<T>` impl + `useAuth` context ¬∑ *Schema/Map:* key scheme.
3. **AC3 ‚Äî Create / update / soft-delete semantics.** `create` sets `id` (typed UUID), audit fields, `version=1`. `update` **requires the caller's `version`**; mismatch ‚Üí `409`. `delete` is soft (`deletedAt` set; row excluded from `list`/`get`).
   *Touches:* `src/shared/data/LocalStorageRepository.ts` ¬∑ *Seam:* `Repository<T>` impl ¬∑ *Schema/Map:* `BaseEntity` versioning (E0-S2).
4. **AC4 ‚Äî Validate-before-persist (Zod).** Every write validates against the entity Zod schema; invalid ‚Üí `422` with field-level details. No invalid row is ever written.
   *Touches:* `src/shared/data/LocalStorageRepository.ts`, `src/shared/domain/schemas.ts` ¬∑ *Seam:* `Repository<T>` impl ¬∑ *Schema/Map:* Zod schemas (`schemas.ts`).
5. **AC5 ‚Äî 4-beat on every mutation (UC-2/UC-3).** Each mutation runs **authorize ‚Üí mutate ‚Üí emit ‚Üí audit**: `can()` gate (`403` in-tenant / `404` out-of-tenant, denial audited `Auth.RoleDenied`) ‚Üí Zod validate + version bump + `canTransition` for state moves (illegal ‚Üí `422`) ‚Üí one `DomainEvent` (canonical type) ‚Üí one immutable `AuditEvent`, all sharing one `correlationId` minted at action start.
   *Touches:* `src/shared/data/LocalStorageRepository.ts` (+ `shared/auth/permissions.ts`, `shared/events/bus.ts`, `shared/events/auditLog.ts`, `shared/events/correlation.ts`) ¬∑ *Seam:* 4-beat use case ¬∑ *Schema/Map:* transition maps (E0-S1) for the state-move path.
6. **AC6 ‚Äî REST-shaped envelopes & codes (NFR-5).** Returns REST envelopes and status codes: `201 + Location` on create, `204` on delete, `200` on read/update, `409`/`422`/`403`/`404` on the error paths; state moves only via the `POST /{id}/transition` shape, never `PATCH status`.
   *Touches:* `src/shared/data/LocalStorageRepository.ts` ¬∑ *Seam:* `Repository<T>` impl ¬∑ *Schema/Map:* n/a.
7. **AC7 ‚Äî Fault-injection toggle (ADR-007).** A `faultInjection` toggle forces error paths (e.g. simulated `409`/`422`/network failure) so optimistic-rollback can be tested deterministically.
   *Touches:* `src/shared/data/faultInjection.ts`, `src/shared/data/LocalStorageRepository.ts` ¬∑ *Seam:* `Repository<T>` impl ¬∑ *Schema/Map:* n/a.

## Inherited Universal Conformance (subset)
- **UC-2 ‚Äî Dual events, one correlationId.** Every mutation emits exactly one `DomainEvent` + one `AuditEvent` on a shared `correlationId`.
- **UC-3 ‚Äî Illegal transition ‚Üí 422.** State moves call `canTransition`; illegal moves return `422`.
- **UC-5 ‚Äî Tenant scope, out-of-tenant ‚Üí 404.** Scope read from auth context only; cross-tenant access returns `404`.
- **TC ‚Äî Traceability.** story ‚Üí spec ‚Üí code ‚Üí test ‚Üí `Closes #`; preview green; `sprint-status.yaml`; passes `bmad-code-review`.
