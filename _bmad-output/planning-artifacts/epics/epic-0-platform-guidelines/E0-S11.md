# E0-S11 — Testing harness + Universal DoD codified

> **GitHub Epic → Feature → Story:** Epic **E0 — Platform Guidelines & Standards** › Feature **0.6 — Conformance, Flags & Testing Harness** › Story **E0-S11**
> **Cut:** Pilot · **Depends on:** E0-S4 (adapter), E0-S6 (guards), E0-S7 (events/audit), E0-S9 (UI states), E0-S10 (flags/fault config)

## Story
As the platform, the full testing stack is wired with the cross-tenant isolation and architecture-fitness gates, and the Universal Conformance ACs + DoD are codified as the inherited story template, so that every Epic 1–5 story ships against one provable standard.

## Implementation-Ready Acceptance Criteria
1. **AC1 — Stack wired (ADR-013).** Vitest + RTL + Playwright are wired with a runnable sample of each test type.
   *Touches:* test config (`vitest`/`playwright`), `src/shared/**/*.test.ts(x)`, `e2e/*` · *Seam:* testing harness · *Schema/Map:* n/a.
2. **AC2 — Cross-tenant isolation E2E (UC-5, SM-P3).** A pilot-level E2E scaffold asserts **0 cross-tenant leaks** against the localStorage adapter.
   *Touches:* `e2e/cross-tenant-isolation.spec.ts` · *Seam:* E2E against `LocalStorageRepository` · *Schema/Map:* key scheme (E0-S4).
3. **AC3 — Optimistic-rollback test (ADR-007).** A test uses the fault-injection toggle to force a mutation failure and asserts the optimistic UI rolls back (`onMutate`/`onError`/`onSettled`).
   *Touches:* `src/shared/data/repository.test.ts` (+ RTL view), `src/shared/data/faultInjection.ts` · *Seam:* optimistic-rollback · *Schema/Map:* n/a.
4. **AC4 — Universal DoD published (TC).** The Universal Conformance ACs (UC-1…UC-5) + DoD checklist are published as the **story template every Epic 1–5 story inherits**.
   *Touches:* the shared story template (this `_bmad-output/planning-artifacts/epics/` standard) · *Seam:* DoD/traceability · *Schema/Map:* n/a.
5. **AC5 — Sprint status + preview gate.** `sprint-status.yaml` and the preview-deploy gate are wired so every story tracks state and previews green before merge.
   *Touches:* `sprint-status.yaml`, CI preview-deploy gate · *Seam:* DoD/traceability · *Schema/Map:* n/a.
6. **AC6 — Architecture-fitness test (NFR-4, SM-M5 pilot proxy).** A test asserts **no `src/features/*` imports `localStorage` or a concrete repository** — features depend only on the `Repository<T>` interface.
   *Touches:* `e2e/architecture-fitness.spec.ts` (or equivalent fitness test) · *Seam:* architecture-fitness · *Schema/Map:* n/a.
