# E0-S5 — AuthProvider + useAuth() + mock SSO

> **GitHub Epic → Feature → Story:** Epic **E0 — Platform Guidelines & Standards** › Feature **0.3 — Auth & RBAC Kernel** › Story **E0-S5**
> **Cut:** Pilot · **Depends on:** E0-S7 (audit bus for auth events)

## Story
As the platform, identity flows through one `AuthProvider` exposing session claims via `useAuth()`, so that tenant/subsidiary scope and roles come from a single seam that later swaps mock SSO for OIDC without touching call sites.

## Implementation-Ready Acceptance Criteria
1. **AC1 — Session claims.** The session exposes claims `{ userId, tenantId, subsidiaryId | null, roles[], exp }`; `useAuth()` returns the current session (and helpers like `isAuthenticated`).
   *Touches:* `src/shared/auth/AuthProvider.tsx`, `src/shared/auth/useAuth.ts` · *Seam:* auth context · *Schema/Map:* claims shape.
2. **AC2 — Tenant-admin roll-up.** `tenant_admin` carries `subsidiaryId = null`, which relaxes the subsidiary filter so the repository returns all tenant rows (roll-up); subsidiary users carry a concrete `subsidiaryId`.
   *Touches:* `src/shared/auth/AuthProvider.tsx` · *Seam:* auth context consumed by `LocalStorageRepository` scoping (E0-S4) · *Schema/Map:* claims shape.
3. **AC3 — Audited auth events (UC-2).** `Auth.LoggedIn`, `Auth.LoginFailed`, `Auth.LoggedOut`, and `Auth.RoleDenied` are emitted to the dual event/audit streams with a `correlationId`.
   *Touches:* `src/shared/auth/AuthProvider.tsx` (+ `shared/events/bus.ts`, `shared/events/auditLog.ts`) · *Seam:* auth context + event/audit bus · *Schema/Map:* canonical `Auth.*` event types.
4. **AC4 — OIDC seam (ADR-009).** `AuthProvider` is the single seam for future OIDC; mock SSO sits behind the same interface so swapping the identity source touches only the provider, not consumers.
   *Touches:* `src/shared/auth/AuthProvider.tsx` · *Seam:* auth context · *Schema/Map:* n/a.

## Inherited Universal Conformance (subset)
- **UC-2 — Dual events, one correlationId.** Auth lifecycle events hit both streams on a shared `correlationId`.
- **TC — Traceability.** story → spec → code → test → `Closes #`; preview green; `sprint-status.yaml`; passes `bmad-code-review`.

- **Modules (NFR-11):** `src/shared/auth/AuthProvider.tsx`, `src/shared/auth/useAuth.ts`.
- **Patterns:** single auth seam; auth context = the only source of tenant/subsidiary scope (feeds NFR-6 two-gate authZ and the repository scoping in E0-S4); correlationId threading.
- **ADRs:** ADR-009 (one `AuthProvider`, mock SSO → OIDC, claims shape, two gates, 404-not-403).

## Test Requirements (NFR-12)
- **Vitest/RTL:** `useAuth()` exposes the full claims set; `tenant_admin` yields `subsidiaryId=null`; login/logout/failed-login/role-denied each emit the correct `Auth.*` event with a `correlationId` (assert via the E0-S7 helper).
- **RTL:** a consumer component rendered inside `AuthProvider` reads claims; rendered outside throws/guards.
- **Playwright:** auth bootstrap is exercised by the cross-tenant isolation E2E (E0-S11).
