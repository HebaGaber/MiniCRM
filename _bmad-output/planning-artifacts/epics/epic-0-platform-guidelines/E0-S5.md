# E0-S5 ‚Äî AuthProvider + useAuth() + mock SSO

> **GitHub Epic ‚Üí Feature ‚Üí Story:** Epic **E0 ‚Äî Platform Guidelines & Standards** ‚Ä∫ Feature **0.3 ‚Äî Auth & RBAC Kernel** ‚Ä∫ Story **E0-S5**
> **Cut:** Pilot ¬∑ **Depends on:** E0-S7 (audit bus for auth events)

## Story
As the platform, identity flows through one `AuthProvider` exposing session claims via `useAuth()`, so that tenant/subsidiary scope and roles come from a single seam that later swaps mock SSO for OIDC without touching call sites.

## Implementation-Ready Acceptance Criteria
1. **AC1 ‚Äî Session claims.** The session exposes claims `{ userId, tenantId, subsidiaryId | null, roles[], exp }`; `useAuth()` returns the current session (and helpers like `isAuthenticated`).
   *Touches:* `src/shared/auth/AuthProvider.tsx`, `src/shared/auth/useAuth.ts` ¬∑ *Seam:* auth context ¬∑ *Schema/Map:* claims shape.
2. **AC2 ‚Äî Tenant-admin roll-up.** `tenant_admin` carries `subsidiaryId = null`, which relaxes the subsidiary filter so the repository returns all tenant rows (roll-up); subsidiary users carry a concrete `subsidiaryId`.
   *Touches:* `src/shared/auth/AuthProvider.tsx` ¬∑ *Seam:* auth context consumed by `LocalStorageRepository` scoping (E0-S4) ¬∑ *Schema/Map:* claims shape.
3. **AC3 ‚Äî Audited auth events (UC-2).** `Auth.LoggedIn`, `Auth.LoginFailed`, `Auth.LoggedOut`, and `Auth.RoleDenied` are emitted to the dual event/audit streams with a `correlationId`.
   *Touches:* `src/shared/auth/AuthProvider.tsx` (+ `shared/events/bus.ts`, `shared/events/auditLog.ts`) ¬∑ *Seam:* auth context + event/audit bus ¬∑ *Schema/Map:* canonical `Auth.*` event types.
4. **AC4 ‚Äî OIDC seam (ADR-009).** `AuthProvider` is the single seam for future OIDC; mock SSO sits behind the same interface so swapping the identity source touches only the provider, not consumers.
   *Touches:* `src/shared/auth/AuthProvider.tsx` ¬∑ *Seam:* auth context ¬∑ *Schema/Map:* n/a.
5. **AC5 ‚Äî Session is established, persisted, and lands the app (DEC-CC-7).** A successful `signIn(roleId)` **sets the session in the auth context** AND **persists it to `sessionStorage`** (`mincrm.auth.session`); on mount the provider **restores** any valid, unexpired persisted session **silently** ‚Äî no `Auth.LoggedIn` re-emit, since a reload is not a new login (a malformed/expired blob is ignored and treated as signed-out). `signOut` clears the persisted session. So a full page reload **keeps the user signed in** instead of returning to `/sign-in`. After sign-in the app **redirects off `/sign-in` to the app home** (`/` ‚Üí `/rollup`; the prototype's "lands on Dashboard"). Storage access is guarded so an unavailable/locked store degrades to in-memory-only, never throws. **Scope note:** this is the auth kernel's own session store (not feature/entity persistence), and the future OIDC swap (AC4) replaces it behind the same seam.
   *Touches:* `src/shared/auth/AuthProvider.tsx`, `src/app/router.tsx` (`SignInPage` redirect) ¬∑ *Seam:* auth context + session persistence ¬∑ *Schema/Map:* `SessionClaims` JSON in `sessionStorage`.

## Test Requirements (NFR-12)
- **Vitest/RTL:** `useAuth()` exposes the full claims set; `tenant_admin` yields `subsidiaryId=null`; login/logout/failed-login/role-denied each emit the correct `Auth.*` event with a `correlationId` (assert via the E0-S7 helper).
- **RTL:** a consumer component rendered inside `AuthProvider` reads claims; rendered outside throws/guards.
- **RTL integration (AC5, DEC-CC-7):** picking a role establishes the session and **lands on the app home** off `/sign-in`; a fresh provider mount (reload) **restores** the persisted session and stays signed in; a silent restore emits **no** `Auth.LoggedIn`. *(Implemented: `src/app/SignInFlow.test.tsx`.)*
- **Playwright:** auth bootstrap is exercised by the cross-tenant isolation E2E (E0-S11).
