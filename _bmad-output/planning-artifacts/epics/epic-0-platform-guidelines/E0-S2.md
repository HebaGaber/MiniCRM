# E0-S2 — Author BaseEntity and canonical entity types

> **GitHub Epic → Feature → Story:** Epic **E0 — Platform Guidelines & Standards** › Feature **0.1 — Domain & Status Kernel** › Story **E0-S2**
> **Cut:** Pilot · **Depends on:** E0-S1 (status literals)

## Story
As the platform, all canonical entities derive from one `BaseEntity` and carry typed IDs and lineage, so that tenancy, audit, versioning, and saga state are uniform across every feature.

## Implementation-Ready Acceptance Criteria
1. **AC1 — `BaseEntity`.** Define `BaseEntity` per NFR-2 with: `id`, `tenantId`, `subsidiaryId | null`, `createdAt`, `updatedAt`, `createdBy`, `updatedBy`, `version` (number), `deletedAt | null` (soft-delete marker). Every entity extends it.
   *Touches:* `src/shared/domain/types.ts` · *Seam:* domain kernel · *Schema/Map:* `BaseEntity`.
2. **AC2 — Canonical entity types.** Define `Tenant`, `Subsidiary`, `User`, `Customer` (shared), `Lead`, and `Ticket`, each extending `BaseEntity` and using the E0-S1 status/role literals. **`Customer` carries `taxRegistrationNumber?: string` and `contactAddress?: string` (DEC-CC-2)** — optional fields that the §3.2 activation gate (E3-S2) requires before `onboarding → active`.
   *Touches:* `src/shared/domain/types.ts`, `src/shared/domain/lead.types.ts`, `src/shared/domain/customer.types.ts`, `src/shared/domain/ticket.types.ts`, `src/shared/domain/tenant.types.ts` · *Seam:* domain kernel · *Schema/Map:* E0-S1 literal unions; `Customer` adds `taxRegistrationNumber?`/`contactAddress?` (DEC-CC-2).
3. **AC3 — Bidirectional conversion lineage (flag G).** `Customer` carries `convertedFromLeadId?: string`; `Lead` carries `convertedToCustomerId?: string`. The pair is the durable lineage written by the conversion saga (E3-S1).
   *Touches:* `src/shared/domain/customer.types.ts`, `src/shared/domain/lead.types.ts` · *Seam:* domain kernel · *Schema/Map:* lineage fields.
4. **AC4 — Type-prefixed UUID helper.** Export an ID factory producing type-prefixed UUIDs (`lead_…`, `cust_…`, `tkt_…`, `wf_…`, plus tenant/subsidiary/user prefixes). All entity creation uses it so IDs are self-describing.
   *Touches:* `src/shared/domain/types.ts` · *Seam:* domain kernel ID factory consumed by repository *create* · *Schema/Map:* n/a.
5. **AC5 — `WorkflowInstance` saga entity.** Define `WorkflowInstance` extending `BaseEntity`: `{ id (wf_…), type, status, currentStep, steps[], completedSteps[], correlationId, payload, ... }`. This is the **persisted state the conversion saga (E3-S1) and onboarding workflow (E3-S2) read and resume from** via the `WorkflowRunner`.
   *Touches:* `src/shared/domain/types.ts` (lives with `BaseEntity`) · *Seam:* domain kernel · *Schema/Map:* `WorkflowInstance`.
