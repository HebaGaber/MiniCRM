# E6-S2 — Swap to HttpRepository at the composition root

> **GitHub Epic → Feature → Story:** Epic **E6 — Server-Side Trust Boundary** › Feature **6.2 — Repository Swap (0-change proof)** › Story **E6-S2**
> **Cut:** DESIGN-ONLY (NOT in the 2-week pilot) · **Depends on:** E0-S3/E0-S4 (the seam) + E6-S1 (the server)

> ## ⚠️ DESIGN-ONLY — NOT IN THE 2-WEEK PILOT CUT (ADR-005)
> Production target, specified as implementation-ready design. The pilot ships on `LocalStorageRepository`; this story swaps to `HttpRepository` against the **production server (E6-S1)** and **proves the swap touches only the composition root + adapter** — zero feature-code change (success metric SM-M5).

## Story
As the team, I prove the repository swap requires **zero feature-code change**, so that the Strangler-Fig seam (ADR-004) is validated end-to-end against the real server.

## Implementation-Ready Acceptance Criteria
1. **AC1 — HttpRepository implements `Repository<T>` against the REST contract.** A new `HttpRepository<T>` implements the full seam — `list(q?)→Page<T>`, `get(id)`, `create`, `update(id,patch,version)`, `remove(id)` — over the REST API (NFR-5): `/api/v1`, `POST /{id}/transition`, `201 + Location`, `204`, `409` (version mismatch), `422` (validation/illegal transition), `404` (out-of-tenant). Honors `Page<T>` defaults (`pageSize=25`, max 100) and threads `correlationId` headers.
   *Touches:* `src/shared/data/HttpRepository.ts` (+ `src/shared/data/Repository.ts` interface, unchanged) · *Seam:* `Repository<T>` REST adapter (NFR-5) · *Schema/Map:* REST status-code contract (`409`/`422`/`404`).
2. **AC2 — Injected at the composition root (Strangler-Fig, ADR-004).** `HttpRepository` is wired in **only** at the composition root; feature code keeps depending on the `Repository<T>` interface, never the concrete class.
   *Touches:* `src/app/composition.ts` · *Seam:* composition-root dependency injection · *Schema/Map:* n/a.
3. **AC3 — ZERO feature-code change (SM-M5).** The swap diff touches **only** `src/app/composition.ts` + the new adapter `src/shared/data/HttpRepository.ts`. **`src/features/*` is untouched**; the diff excludes `src/app` (other than composition.ts) and `src/shared/data` (other than the new adapter). Enforced by the **architecture-fitness test** (ADR-004) which fails if any `src/features/*` file changes as part of the swap.
   *Touches:* `src/app/composition.ts`, `src/shared/data/HttpRepository.ts` (and nothing under `src/features/*`) · *Seam:* architecture-fitness guard · *Schema/Map:* n/a.
4. **AC4 — Feature tests pass unchanged.** The existing Vitest/RTL feature test suites pass **without modification** against the `HttpRepository` (mocked/contract-tested transport), proving behavioral parity with `LocalStorageRepository`.
   *Touches:* existing `src/features/*` tests (run unchanged) · *Seam:* `Repository<T>` contract tests · *Schema/Map:* shared `repository.test.ts` contract suite.

## Test Requirements (NFR-12)
- **Vitest:** `HttpRepository` contract tests against the shared `Repository<T>` suite (same suite that `LocalStorageRepository` passes), incl. `409` version-mismatch, `422` validation/illegal-transition, `404` out-of-tenant, `Page<T>` pagination bounds.
- **RTL:** existing feature component tests run **unchanged** (four states, optimistic rollback via fault injection) and pass against the swapped adapter.
- **Playwright:** `architecture-fitness.spec.ts` asserts the swap diff touches only `composition.ts` + `HttpRepository.ts` and **no `src/features/*` file** (SM-M5 0-change proof).
