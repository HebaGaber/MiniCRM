# E6-S3 — Cross-tenant isolation E2E (the acceptance gate)

> **GitHub Epic → Feature → Story:** Epic **E6 — Server-Side Trust Boundary** › Feature **6.3 — Cross-Tenant Isolation Gate** › Story **E6-S3**

> ## ⚠️ DESIGN-ONLY — NOT IN THE 2-WEEK PILOT CUT (ADR-005)
> Production target, specified as implementation-ready design. **This is the REAL security boundary.** It is distinct from the **E0-S11 pilot proxy**, which is **logical-only** (it proves the client adapter keys data correctly) and is **NOT a security boundary**. E6-S3 is the **definitive 0-leak gate** against the real server + Postgres RLS.

## Story
As the platform, I prove **isolation before production ships**, so that no tenant can ever read, write, or infer another tenant's data.

## Implementation-Ready Acceptance Criteria
1. **AC1 — 0 cross-tenant leaks across ALL entities (SM-P3).** A Playwright E2E, authenticated as tenant A, attempts to access tenant B records across **every entity** (Lead, Customer, Ticket, Tenant/subsidiary, and any read-model/widget surface) via list, get, update, transition, remove, and direct-ID URL manipulation — and asserts **zero leaks** (SM-P3). Includes subsidiary-scope checks (subsidiary user cannot reach sibling-subsidiary rows; `subsidiary_id = null` rules honored per ADR-002).
   *Touches:* `e2e/cross-tenant-isolation.spec.ts` · *Seam:* full stack — swapped `HttpRepository` → REST → server authZ → Postgres RLS · *Schema/Map:* all entity routes under `/api/v1`.
2. **AC2 — This E2E is the GATE for Epic 6 sign-off (ADR-005).** `cross-tenant-isolation.spec.ts` is a **required, blocking** check: Epic 6 is **not signed off and production does not ship** unless it is green. Wired into CI as a hard gate (not advisory).
   *Touches:* CI pipeline config + `e2e/cross-tenant-isolation.spec.ts` · *Seam:* CI gate · *Schema/Map:* n/a.
3. **AC3 — Cross-tenant access returns 404.** Every attempt to read/mutate another tenant's record returns **404** (not 403, not 200-with-empty) — existence is never disclosed (ADR-005, NFR-6). The test asserts the **404 status** and the absence of any leaked field in the body/headers.
   *Touches:* `e2e/cross-tenant-isolation.spec.ts` (assertions) · *Seam:* server authZ + RLS · *Schema/Map:* n/a.
