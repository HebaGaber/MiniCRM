# E2-S4 — Lead detail + activity timeline

> **GitHub Epic → Feature → Story:** Epic **E2 — Leads: Capture → Qualify** › Feature **2.3 — Qualification State Machine** › Story **E2-S4**
> **Cut:** Pilot · **Depends on:** E2-S1 (lead create), E2-S3 (transition mutation) + Epic 0 kernels (E0-S6 action guard, E0-S7 events, E0-S9 DetailPage/StatusPill) + E1-S1 (scope)

## Story
As a salesperson, I see the lead's full history and act on it, so that I can decide and move the lead with all its context in one place.

## Implementation-Ready Acceptance Criteria
1. **AC1 — DetailPage with StatusPill; edit + transition via guarded mutations.** The lead renders through the shared `DetailPage` template with a `StatusPill` (tone from `STATUS_TONE`). Edit and transition actions invoke the **guarded mutations** (E0-S6 action guard via `can()`), so each runs the full 4-beat — every edit emits `Lead.Updated` + audit and every status move emits `Lead.StatusChanged` + audit (UC-2), and any status move obeys `LEAD_TRANSITIONS` (UC-3, illegal → `422`, pill unchanged). Transitions go through `POST /leads/{id}/transition`, never `PATCH status`.
   *Touches:* `src/features/leads/{LeadDetailPage.tsx,lead.service.ts}`, `src/shared/ui/templates/DetailPage`, `src/shared/ui/components/StatusPill`, `src/shared/auth/permissions.ts`, `src/shared/domain/status.ts` · *Seam:* 4-beat (authorize→mutate→emit→audit) · *Schema/Map:* `leadUpdateSchema` (Zod); `LEAD_TRANSITIONS`.
2. **AC2 — Activity timeline = DomainEvent stream (audit-as-feature), record-view gated (ADR-016).** An `ActivityTimeline` renders the lead's domain events (`Lead.Created`/`Updated`/`StatusChanged`, etc.) in time order. Per **ADR-016** it reads the `DomainEvent` stream filtered by `entityType="Lead" + entityId`, gated by **permission to view the record** (so `viewer` within scope is included). This is the **per-record product surface — DISTINCT** from the matrix-gated raw Audit/Events log (`AuditEvent` with before/after) of **E4-S4**. Built on the shared `ActivityTimeline` pattern reused in `features/tickets`.
   *Touches:* `src/features/leads/ActivityTimeline.tsx`, `src/shared/events/bus.ts`, `src/shared/auth/permissions.ts` · *Seam:* `DomainEvent` stream read (filtered by `entityType+entityId`), record-view permission gate · *Schema/Map:* canonical event registry; n/a transition.
3. **AC3 — Four UI states (UC-1).** Both the detail body and the timeline render loading/empty/error/data via `<QueryStateBoundary>`; a lead with no activity yet shows the timeline's empty state (not an error); action mutations are optimistic with rollback on injected fault.
   *Touches:* `src/features/leads/{LeadDetailPage.tsx,ActivityTimeline.tsx}`, `src/shared/ui/QueryStateBoundary.tsx` · *Seam:* four-state boundary + optimistic rollback · *Schema/Map:* n/a.

## Inherited Universal Conformance (subset)
- **UC-1 — Four UI states.** Detail body + timeline render via `<QueryStateBoundary>`; empty timeline ≠ error.
- **UC-2 — Dual events one correlationId.** Each edit/transition from detail emits its domain event + one `AuditEvent` under one `correlationId`.
- **UC-3 — Illegal transition 422.** Status moves from detail obey `LEAD_TRANSITIONS`; illegal → `422`, pill unchanged.
- **UC-5 — Tenant scope.** Detail/timeline only for a lead in the caller's scope; out-of-tenant id → 404; timeline gated by record-view permission within scope (ADR-016).
- **TC — Traceability.** story → spec → code → test → `Closes #`; preview green; `sprint-status.yaml`; passes `bmad-code-review`.

## Test Requirements (NFR-12)
- **Vitest:** `ActivityTimeline` data layer returns the lead's `DomainEvent`s filtered by `entityType+entityId` in time order; record-view permission gate allows `viewer` within scope and denies out-of-scope; edit mutation emits `Lead.Updated` + audit (one correlationId); transition mutation reuses E2-S3 4-beat behavior.
- **RTL:** `LeadDetailPage` renders `StatusPill` (tone from `STATUS_TONE`) and four states; timeline shows events in time order and an empty state for a fresh lead; an illegal transition from the detail leaves the pill unchanged and surfaces an error; edit/transition are optimistic with rollback on injected fault.
- **Playwright:** open a lead detail, edit a field (timeline gains a `Lead.Updated` entry), perform a legal transition (timeline gains `Lead.StatusChanged`, pill updates), attempt an illegal transition (`422`, pill unchanged); confirm a `viewer` in scope can read the timeline.
