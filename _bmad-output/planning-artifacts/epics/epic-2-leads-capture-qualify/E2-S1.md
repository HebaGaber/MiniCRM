# E2-S1 — Capture a lead

> **GitHub Epic → Feature → Story:** Epic **E2 — Leads: Capture → Qualify** › Feature **2.1 — Lead Capture** › Story **E2-S1**
> **Cut:** Pilot · **Depends on:** Epic 0 kernels (E0-S4 repository/4-beat, E0-S6 action guard, E0-S7 events, E0-S9 EntityForm/UI) + E1-S1 (tenant scope)

## Story
As a salesperson, I capture a lead with a source and a single owner, so that every new prospect enters the pipeline in a known starting state with a clear point of accountability.

## Implementation-Ready Acceptance Criteria
1. **AC1 — Capture is sales/tenant_admin only (UC-5 + matrix).** The create action is authorized via the `can()` guard for `lead`/`create`; `sales` and `tenant_admin` are granted, all other roles denied (deny-wins). Denial in-tenant → `403`; out-of-tenant scope → `404`; the denial is audited as `Auth.RoleDenied`.
   *Touches:* `src/features/leads/lead.service.ts`, `src/shared/auth/permissions.ts`, `src/shared/data/LocalStorageRepository.ts` · *Seam:* 4-beat **authorize** (`can()`) · *Schema/Map:* permission matrix (ADR-015); n/a transition.
2. **AC2 — Zod-validated EntityForm; invalid → 422 + field details (flag J).** Capture renders through the shared `EntityForm` template (RHF v7 + Zod v4); the `leadCreateSchema` validates before persist. Invalid input resolves as `422` carrying per-field detail mapped back onto the form fields; nothing is persisted on failure.
   *Touches:* `src/features/leads/LeadForm.tsx`, `src/shared/domain/schemas.ts`, `src/shared/ui/templates/EntityForm` · *Seam:* 4-beat **mutate** (validate-before-persist) · *Schema/Map:* `leadCreateSchema` (Zod).
3. **AC3 — Source enum + exactly one owner.** `source ∈ {web, referral, event, outbound, import}` and exactly one `ownerId` are required by the schema; absent/invalid source or missing/multiple owner → `422`. `ownerId` is a single reference (not an array).
   *Touches:* `src/shared/domain/lead.types.ts`, `src/shared/domain/schemas.ts`, `src/features/leads/LeadForm.tsx` · *Seam:* 4-beat **mutate** · *Schema/Map:* `leadCreateSchema` (source enum + single `ownerId`).
4. **AC4 — Creates Lead in `new`, emits Lead.Created + audit (UC-2).** A successful capture creates a `Lead` (BaseEntity + tenant/subsidiary from context) with `status = "new"`, returns `201 + Location`, and in the same 4-beat **emits** `Lead.Created` (canonical type) and **audits** one immutable `AuditEvent`, threaded by one `correlationId` minted at action start.
   *Touches:* `src/features/leads/lead.service.ts`, `src/shared/data/LocalStorageRepository.ts`, `src/shared/events/{bus.ts,auditLog.ts,correlation.ts}` · *Seam:* 4-beat **emit** + **audit** · *Schema/Map:* status seed = `new` (`LEAD_TRANSITIONS` start state).
5. **AC5 — BANT as a free judgment note (flag J).** BANT (budget/authority/need/timeline) is captured as a single free-text judgment note field on the lead — **no scoring engine, no computed qualification value**. It is persisted verbatim and never gates any transition.
   *Touches:* `src/shared/domain/lead.types.ts`, `src/shared/domain/schemas.ts`, `src/features/leads/LeadForm.tsx` · *Seam:* 4-beat **mutate** · *Schema/Map:* `leadCreateSchema` (optional free-text `bantNote`).
6. **AC6 — Four UI states (UC-1).** The capture surface and its post-create redirect render loading/empty/error/data via `<QueryStateBoundary>`; submit uses the optimistic mutation contract with rollback on error (fault-injection toggle exercisable).
   *Touches:* `src/features/leads/LeadForm.tsx`, `src/shared/ui/QueryStateBoundary.tsx` · *Seam:* optimistic mutation (`onMutate/onError/onSettled`) · *Schema/Map:* n/a.

## UX & Behavior (from prototype)
*Source: `prototype/app/leads.jsx` (`LeadForm`). Build identical to the prototype.*

- **`LeadForm` modal — fields in exact order:** **Name** (required) · **Company** · **Email** (format-validated *if present*) · **Phone** · **Source** (required select: Web / Referral / Event / Outbound / Import) · **Owner** (required select — one of the four lead owners) · **BANT note** (free-text textarea; help text *"budget, authority, need, timing. Free text, not a score."*).
- **Validation:** submit **disabled until valid**; on a failed submit **focus jumps to the first errored field**; inline field errors with an `alert-circle` icon.
- **Create is optimistic** → status `new`; success toast **"Lead captured · <name> created in status 'new'"**, then **redirect to the new lead's detail**. **OutcomePicker** "Server error" path removes the draft after ~700ms (snap-back `--crm-slow`) and fires a **persistent rollback toast + Retry**.
- **Edit** updates in place, appends an *"updated the lead details"* timeline event, and shows a success toast.
- Focus-trap + first-field autofocus + `Esc` cancels.
- **Motion:** modal enter `crm-pop` `--crm-base`; optimistic apply `--crm-instant`; rollback `--crm-slow`; toast `--crm-fast`.

- **UC-5 — Tenant scope.** Lead is created in the caller's tenant/subsidiary from auth context only; capture is role-gated to sales/tenant_admin.
