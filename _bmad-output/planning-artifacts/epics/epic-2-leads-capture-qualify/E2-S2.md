# E2-S2 вЂ” List and filter leads

> **GitHub Epic в†’ Feature в†’ Story:** Epic **E2 вЂ” Leads: Capture в†’ Qualify** вЂє Feature **2.2 вЂ” Lead List & Filter** вЂє Story **E2-S2**
> **Cut:** Pilot В· **Depends on:** E2-S1 (leads to list) + Epic 0 kernels (E0-S4 repository, E0-S9 ListPage/DataTable/StatusPill) + E1-S1 (tenant scope)

## Story
As a salesperson, I find leads by status and owner, so that I can work my pipeline without scrolling through every record.

## Implementation-Ready Acceptance Criteria
1. **AC1 вЂ” ListPage with DataTable + StatusPill.** The leads list renders through the shared `ListPage` template using `DataTable`; each row's status renders via `StatusPill` whose tone is read **only** from `STATUS_TONE` (no hex/px вЂ” NFR-10). Columns include name/source/owner/status.
   *Touches:* `src/features/leads/LeadsListPage.tsx`, `src/shared/ui/templates/ListPage`, `src/shared/ui/components/{DataTable,StatusPill}` В· *Seam:* `Repository<T>.list` В· *Schema/Map:* `STATUS_TONE` (StatusPill tone).
2. **AC2 вЂ” Filter, search, sort, pagination (Page<T>).** Filter by `status` and `ownerId`, free-text `q`, sortable columns, and pagination via `Repository<T>.list(q?) в†’ Page<T>` (default `pageSize=25`, max 100). Filters/sort/page are passed as query params and reflected in the URL.
   *Touches:* `src/features/leads/{LeadsListPage.tsx,lead.service.ts}`, `src/shared/data/{Repository.ts,LocalStorageRepository.ts}`, `src/shared/ui/components/{FilterBar,Toolbar}` В· *Seam:* `Repository<T>.list` (query в†’ `Page<T>`) В· *Schema/Map:* n/a (read path).
3. **AC3 вЂ” Scoped to tenant/subsidiary (UC-5).** The list is scoped from auth context only via the repository scope seam (ADR-002): `tenant_admin` (subsidiaryId=null) sees all tenant rows; a subsidiary user sees own + parent-level (`subsidiaryId=null`) only. No scope parameter is accepted from the caller.
   *Touches:* `src/shared/data/LocalStorageRepository.ts`, `src/features/leads/lead.service.ts` В· *Seam:* `Repository<T>.list` scope filter В· *Schema/Map:* n/a.
4. **AC4 вЂ” Four UI states incl. empty-with-primary-action (UC-1).** The list renders loading/empty/error/data via `<QueryStateBoundary>`; the **empty** state offers a primary "Capture a lead" action (deep-link to E2-S1), not a bare message; the error state is retryable.
   *Touches:* `src/features/leads/LeadsListPage.tsx`, `src/shared/ui/QueryStateBoundary.tsx`, `src/shared/ui/components/EmptyState` В· *Seam:* four-state boundary В· *Schema/Map:* n/a.
