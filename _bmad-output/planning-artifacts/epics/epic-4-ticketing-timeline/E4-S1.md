# E4-S1 вЂ” Create a ticket against an eligible customer

> **GitHub Epic в†’ Feature в†’ Story:** Epic **E4 вЂ” Ticketing & Activity Timeline** вЂє Feature **4.1 вЂ” Ticket Creation (customer-state gated)** вЂє Story **E4-S1**

## Story
As a support agent, I open a ticket against a customer, but only when the customer is ready, so that support work only attaches to live accounts and never to a prospect, inactive, or churned record.

## Implementation-Ready Acceptance Criteria
1. **AC1 вЂ” Role gate: ticket CREATE is granted to every role (DEC-CC-1); edit/assign stays support/tenant_admin.** Per the ratified amendment **DEC-CC-1** (В§6.2), the route guard shows ticket-create and the action guard permits the create mutation for **`tenant_admin`, `sales`, `support`, and `viewer`** вЂ” tickets are the one entity anyone in scope may raise. **Ticket edit/assign remains support/tenant_admin** (sales/viewer read) вЂ” that gate is unchanged. (Supersedes the prior "support/tenant_admin only; flag F" gate; conflict C-1 resolved *adopt prototype*.) The customer-state precondition (AC2) and scope checks still apply to every creator.
   *Touches:* `src/features/tickets/*` (create surface), `src/shared/auth/permissions.ts`, `src/shared/auth/guards.tsx` В· *Seam:* two-gate authZ + 4-beat authorize В· *Schema/Map:* В§2.2 matrix (Tickets **create** row = all roles; edit/assign row unchanged).
2. **AC2 вЂ” Customer-state gate as an action-guard precondition (UJ-4 edge case в†’ 422).** Ticket creation is allowed **only when the linked customer is `active` or `onboarding`**; for `prospect`/`inactive`/`churned` the create is **blocked** as an **action-guard precondition** and rejected with **`422`** (not silently allowed) вЂ” pill/state unchanged. The gate reads `CustomerStatus` from the customer record inside the repository (scope-resolved per UC-5).
   *Touches:* `src/features/tickets/*` (create use case), `src/shared/domain/customer.types.ts` (read `CustomerStatus`), `src/shared/auth/permissions.ts` (precondition) В· *Seam:* 4-beat authorize beat (precondition guard) В· *Schema/Map:* eligible-state set `{active, onboarding}`; `422` on violation.
3. **AC3 вЂ” Links to exactly one Customer; inherits tenant/subsidiary.** A ticket carries exactly one `customerId` and **inherits the customer's `tenantId`/`subsidiaryId`** вЂ” the ticket is created in the customer's scope (never a caller-supplied scope, UC-5); out-of-scope `customerId` resolves not-found (`404`).
   *Touches:* `src/shared/domain/ticket.types.ts`, `src/shared/data/LocalStorageRepository.ts` В· *Seam:* `Repository<T>.create` scope resolution В· *Schema/Map:* `Ticket` Zod schema (`customerId` required, single).
4. **AC4 вЂ” Priority vocabulary; severity is not a field.** `priority в€€ low | medium | high | urgent` (`TicketPriority` literal). **Severity is vocabulary only вЂ” not a field** on `Ticket`.
   *Touches:* `src/shared/domain/ticket.types.ts`, `src/shared/domain/schemas.ts` В· *Seam:* Zod validate-before-persist В· *Schema/Map:* `TicketPriority` literal in `Ticket` Zod schema.
5. **AC5 вЂ” Created in `open`; emits Ticket.Created + audit (UC-2).** A valid create produces a `Ticket` in initial status **`open`** and, via the 4-beat, emits exactly one `Ticket.Created` DomainEvent + one AuditEvent sharing one `correlationId`. Invalid input в†’ `422` + field details.
   *Touches:* `src/features/tickets/*`, `src/shared/data/LocalStorageRepository.ts`, `src/shared/events/{bus.ts,auditLog.ts}` В· *Seam:* 4-beat emit + audit В· *Schema/Map:* `Ticket` Zod schema; initial `open`.
6. **AC6 вЂ” Four ticket states defined (UC-1 vocabulary).** The four-state vocabulary `open / in_progress / pending / resolved / closed` (`TicketStatus`) is the single source from `shared/domain/status.ts`; the create surface renders the four UI states via `<QueryStateBoundary>`.
   *Touches:* `src/shared/domain/status.ts`, `src/features/tickets/*`, `src/shared/ui/QueryStateBoundary.tsx` В· *Seam:* four-state boundary В· *Schema/Map:* `TicketStatus` literal (state vocabulary owned here, transitions in E4-S2).
