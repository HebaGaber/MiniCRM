## 2026-06-06 — Open flags resolved in this PRD

- **F-2 (data residency grain) — RESOLVED (recommended default, pending customer confirmation):** **tenant-grain residency**. Rationale: tenant-grain keeps the Pool (shared Postgres + RLS) model intact (ADR-001/010); subsidiary-grain residency would fracture the two-level Pool and force per-subsidiary placement. Recorded as the recommended default per user instruction; carried to architecture as `[CONFIRM with customer]` — flips to subsidiary-grain only if a customer contract demands per-subsidiary data placement. Logged against ADR-010.
- **F-6 (production DB) — RESOLVED (confirmed):** **PostgreSQL**, because Row-Level Security is the chosen isolation primitive (ADR-001/003) and RLS is Postgres-native. **Boilerplate-mandate check:** no iSolution Platform Boilerplate DB mandate is recorded in any settled artifact; flagged for architecture to confirm no future mandate conflicts before the production build (Epic 6). If a mandate surfaces that is not Postgres, ADR-001's RLS approach must be revisited.
